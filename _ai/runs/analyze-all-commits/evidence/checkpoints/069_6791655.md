## 核心变更
1. **MaterialPropertyBlock 系统**: 新增 `MaterialPropertyBlock.h/cpp`，实现 PerDraw 数据与 PerMaterial 数据分离
2. **PerDraw 分配器**: 新增 `D3D12PerDrawAllocator` 和 `PerDrawAllocator.h`，管理大型 PerDrawBuffer
3. **Material Instantiate**: 新增材质拷贝方法，支持运行时创建材质变体
4. **MeshRenderer 重构**: 适配新的 PerDraw 数据绑定流程
5. **Culling 更新**: `Culling.cpp` 适配新的渲染数据结构

## 风险点
1. **D3D12PerDrawAllocator 新实现**: 复杂的 GPU 内存分配器，可能出现内存泄漏或越界访问
2. **PerDraw 数据生命周期**: PerDrawBuffer 的分配/释放时机与 DrawCall 提交不匹配
3. **Material Instantiate 引用管理**: 拷贝材质的引用计数和资源生命周期管理
4. **接口复杂度**: RenderAPISetMaterial 功能过于复杂，混合了多种数据设置

## 可能的错误原因
1. **PerDrawBuffer 耗尽**: 大型场景下固定大小的 PerDrawBuffer 溢出
2. **绑定状态不一致**: MaterialPropertyBlock 与 Material 的绑定状态未正确同步
3. **Instantiate 材质共享问题**: 拷贝材质修改影响原始材质
4. **D3D12 资源屏障缺失**: PerDrawBuffer 的 GPU 可见性转换未正确处理
