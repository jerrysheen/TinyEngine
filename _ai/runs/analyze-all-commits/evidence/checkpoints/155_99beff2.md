# Checkpoint 24: 99beff2

## Commit 标题
D： 第一版本Bindless调通，后续有位置问题需要修复。主要做的修改是修复Bindless相关的接口， 比如Material和MaterialInstance的设置， 然后GPUScene中接入相关的matIndex数据

## 核心改动
1. **StandardPBR Vertex Pulling Shader Bindless实现**：
   - 取消g_Textures数组的注释，正式启用Bindless纹理采样，支持最多1024个纹理同时采样
   - 新增Tangent属性支持，在顶点着色器中传递切线数据，像素着色器中实现切线空间法线贴图计算
   - 实现完整的PBR光照计算逻辑，包括漫反射、高光反射、法线贴图、金属度/粗糙度采样
   - 通过matIndex从PerMaterialData缓冲区获取材质属性和纹理索引，使用Bindless索引采样对应的纹理

2. **材质系统Bindless接口扩展**：
   - 新增`Material::FlushBindlessIndicesToInstance()`方法，将材质关联的纹理的Bindless索引同步到材质实例数据中
   - 上传材质数据到GPU前自动调用FlushBindlessIndicesToInstance，确保纹理索引正确更新
   - `MaterialInstance`新增`SetValueAtOffset()`接口，支持按偏移直接设置实例数据，用于填充纹理索引
   - 修复`MaterialLayout::AddTextureToBlockOffset()`中的断言错误，将`< 0`改为`<= 0`，确保纹理索引映射正确添加

3. **材质数据结构扩展**：
   - `PerMaterialData`结构体新增NormalTextureIndex、MetallicTextureID、EmissiveTextureID字段，支持更多纹理类型
   - `LoadPerMaterialData`函数更新，读取新增的纹理索引字段
   - StandardPBR材质布局新增NormalTextureID、MetallicTextureID、EmissiveTextureID属性，并建立纹理名称到缓冲区偏移的映射

4. **GPUScene材质索引接入**：
   - 在渲染对象数据处理时，获取材质对象并将材质的分配偏移赋值给PerObjectData的matIndex字段
   - 修复AABB数据上传错误，原来上传的是PerObjectData地址，现在改为上传正确的世界空间包围盒数据
   - 修复脏标记判断逻辑，当MaterialDirty标记存在时也会触发PerObjectData更新，确保材质索引变更及时同步到GPU

## 风险点
1. 注释中提到"后续有位置问题需要修复"，说明当前实现存在纹理坐标或者UV相关的潜在问题
2. Bindless纹理索引同步逻辑如果处理不当，会导致材质采样错误的纹理，出现渲染异常
3. PerMaterialData结构扩展后，需要确保所有使用该结构的地方都正确处理了新字段
4. GPUScene中直接获取ResourceManager的材质资源，如果材质还未加载完成，可能导致空指针访问

## 推测错误原因
在GPU Driven Rendering架构下，传统的材质绑定方式效率低下，每个材质切换都需要绑定资源，无法支持大规模场景渲染；通过Bindless纹理技术，所有纹理都可以通过索引直接访问，不需要逐材质绑定，大幅提升渲染效率；本次提交是Bindless渲染的第一个可用版本，实现了从材质数据准备到Shader采样的完整流程，但还有部分细节问题（比如纹理坐标位置）需要后续修复。
