## 核心修改
### 1. Editor GUI 走渲染线程、与场景渲染同序
- 新增 **`RenderOp::kIssueEditorGUIDraw`** 与空 payload；`Renderer::OnDrawGUI()` 向队列 **PushBlocking** 该指令。
- 主线程在 **`Game::Render`** 里、**`RenderEngine::Render` 之前** 调用 `EditorGUIManager::Update()` → 实质只发「本帧要画 ImGui」标记。
- 渲染线程在 **`kEndFrame`** 中：先 **`RenderAPISubmit()`**（原 EndFrame 里提交/同步部分），再在 **EDITOR** 下若 `hasDrawGUI` 则依次 **`BeginFrame → Render → EndFrame`**（ImGui），最后 **`RenderAPIPresentFrame()`**（`Present` + 轮换 backbuffer + `ResetFrameAllocator` 等从原 EndFrame 拆出）。
- `D3D12EditorGUIManager` 去掉空的 `Update()`；基类 **`EditorGUIManager::Update()`** 仍通过 `RenderEngine::OnDrawGUI()` 发 payload。

### 2. API 拆分：Submit vs Present
- `RenderAPIEndFrame` 拆为 **`RenderAPISubmit`** 与 **`RenderAPIPresentFrame`**，便于在二者之间插入 ImGui 录制与编码。

## 涉及模块

## 风险点
1. **顺序敏感**：ImGui 必须在 submit 之后、present 之前，否则与 swapchain/命令列表生命周期错位。
2. **双线程**：主线程只发标记，真正 ImGui 在渲染线程执行，需注意与输入/焦点时序。

## 可能的错误源
- `hasDrawGUI` 未在合适时机清零或与 `kEndFrame` 配对错误导致偶发不画或重复画。
