### 1. 新增 MaterailRenderState::Reset() 方法
在 `RenderCommand.h` 中为 `MaterailRenderState` 结构体添加重置方法：
```cpp
void Reset()
{
    shaderInstanceID = 0;
    rootSignatureKey;
    enableDepthTest = true;
    enableDepthWrite = true;
    depthComparisonFunc = DepthComparisonFunc::LEQUAL;
    enableBlend = false;
    srcBlend = (BlendState)0;
    destBlend = (BlendState)0;
    hashID = 0;
}
```

### 2. 材质状态缓存优化
在 `GPUSceneRenderPass.h/cpp` 中：
- 将 `m_LastMatState` 从 public 改为 private
- 在 `Execute()` 开始时调用 `m_LastMatState.Reset()`
- 添加材质状态比较逻辑：只有当材质状态 Hash 变化时才重新绑定

### 3. 减少不必要的状态设置

- 现在：只有材质状态变化时才调用上述方法

### 4. 临时限制循环
添加 `if (count != 0) break;` 限制只处理第一个 draw call（待后续优化）

## 风险点
1. **临时 break 语句**：当前只处理第一个 draw call，可能导致部分物体不渲染
2. **状态比较依赖 Hash**：如果 Hash 计算有误，可能导致状态更新遗漏
3. **多材质场景**：需要确保材质切换时状态正确重置

## 可能引入的 Bug
1. 多个不同材质的物体可能无法正确渲染（因为 break 限制）
2. 材质状态缓存可能导致状态更新不及时
3. Reset() 方法中的默认值可能与实际需求不匹配

## 建议调查范围
- 确认 break 语句是否为临时调试代码
- 验证材质状态 Hash 的正确性
