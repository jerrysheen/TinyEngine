## 风险点
1. **多采样器 / 非 s0**：若后续材质或 shader 使用 `s1+` 或每材质不同采样状态，静态 `s0` 无法满足，需恢复动态采样器路径或扩展 `StaticSampler` 数组。
2. **与 bindless / 堆采样器策略冲突**：若工程其它处假设采样器来自 descriptor heap，此提交会与之不一致。
3. **BOM**：PCH 行首 BOM 在极少数编译器/预编译头场景下可能引发告警，宜后续去掉。

## 可能的错误源
- 动态采样器 descriptor table **未正确绑定或未执行** `SetGraphicsRootDescriptorTable`，导致「采样器未设置」类渲染异常；本提交用静态采样器绕过。
- `samplerCount` 与 Root Parameter 布局不一致导致根签名与 PSO / shader 反射不匹配。
