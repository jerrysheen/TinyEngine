# Checkpoint 032: a1265f1

## Commit 标题
D：调整渲染资产设置，改为相机配置， 改Desc配置方式为DescTable
注意DescHeap设置， 材质CPU单独一份，需要的时候Copy到GPU对应位置。

## 核心改动
1. **描述符管理优化**：
   - 重构描述符配置方式，改为使用Descriptor Table（描述符表）进行资源绑定，替代之前逐个绑定描述符的方式，提升绑定效率，符合D3D12的最佳实践
   - 优化描述符分配器和管理器，新增描述符表管理功能，支持CBV/SRV/UAV描述符表的批量分配、拷贝、更新
   - 实现材质参数的CPU/GPU分离存储：CPU端保存所有材质参数的完整副本，需要的时候批量拷贝到GPU对应的常量缓冲区位置，减少GPU内存占用，提升参数更新效率
   
2. **渲染管线扩展**：
   - 新增 `Assets/Shader/BlitShader.hlsl`（+34行），Blit专用Shader，用于将离屏渲染结果拷贝到BackBuffer或者其他渲染目标
   - 新增 `Runtime/Renderer/RenderPipeLine/FinalBlitPass.h/cpp`（+26/+31行），最终Blit Pass实现，负责将前向渲染Pass的离屏结果拷贝到交换链的BackBuffer，完成最终的画面输出
   - 优化前向渲染Pass，现在渲染到离屏FBO，而不是直接渲染到BackBuffer，为后续实现后处理效果打下基础
   
3. **系统适配优化**：
   - 调整渲染资产配置方式，统一由相机进行配置，每个相机可以独立配置自己的渲染资产和渲染流程
   - 大幅优化 `Runtime/Platforms/D3D12/D3D12RenderAPI.cpp`（+259-行），适配新的描述符表绑定方式，优化资源绑定逻辑，提升渲染效率
   - 优化材质参数更新逻辑，支持批量更新多个材质参数，减少API调用次数
   - 优化FrameBuffer管理器，支持Blit操作接口，可以在不同渲染目标之间拷贝纹理数据

## 涉及模块

## 风险点
- 描述符表的布局如果和Shader中的绑定槽不匹配，会导致资源读取错误，渲染异常
- 材质参数的CPU/GPU同步逻辑如果有错误，会导致参数更新不及时，或者更新错误
- 描述符堆如果没有正确管理，会导致描述符溢出，渲染失败
- FinalBlitPass的纹理采样如果配置不正确，会导致画面模糊、拉伸或者颜色失真
- 离屏渲染到Blit到BackBuffer的流程如果有同步问题，会导致画面闪烁或者撕裂

## 推测错误原因
之前的描述符绑定方式效率低，不符合D3D12的最佳实践，而且渲染直接输出到BackBuffer，无法支持后处理等高级特性。本次提交重构了描述符管理系统，采用描述符表进行资源绑定，提升了渲染效率，同时新增了FinalBlitPass，将渲染流程改为离屏渲染+Blit输出的结构，为后续实现后处理、HDR、色调映射等高级渲染特性打下了基础。同时优化了材质参数的存储和更新方式，提升了材质系统的性能。
